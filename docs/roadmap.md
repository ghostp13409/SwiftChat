# SwiftChat Roadmap & Strategy

The development of SwiftChat is divided into six logical phases to ensure a premium, secure, and user-centric experience.

## Phase Overview

| Phase | Title | Focus | Primary Goal |
|---|---|---|---|
| **1** | **Secure Foundation** | Networking & Security | Two devices can talk securely. (COMPLETE) |
| **2** | **Gossip Engine** | Mesh & Database | Messages survive disconnection/replays. (COMPLETE) |
| **3** | **Visual Identity** | Branding & Design | Establish the "Playful Organic" design system. |
| **4** | **Explore UI** | Discovery & UX | Visualizing the "Radar" and Peers. |
| **5** | **Social Features** | Groups & Files | Rich interaction and media sharing. |
| **6** | **Privacy & Polish** | Safety & Performance | Battery optimization and user safety controls. |

## Milestones

### Milestone 1: The "Hello Mesh" (End of Phase 1) ✅
- [x] Profile created and stored.
- [x] Discovery finds a nearby device.
- [x] Encrypted handshake successful.
- [x] Direct text message sent and received.

### Milestone 2: The "Ghost Message" (End of Phase 2) ✅
- [x] Migration to Isar NoSQL database.
- [x] Message survives app restart.
- [x] Message relayed through a 3rd device successfully.
- [x] TTL (24h) auto-deletion verified.

### Milestone 3: The "Brand Soul" (End of Phase 3)
- [ ] Theme Engine with Light/Dark Neubrutalist support.
- [ ] Custom "Physical" component library (Buttons, Cards).
- [ ] Premium Onboarding flow with P2P permission guidance.

### Milestone 4: The "Radar Experience" (End of Phase 4)
- [ ] Radar view showing real-time proximity bubbles.
- [ ] Profile cards with topics and match scores.
- [ ] Playful "Handshake" connection animations.

### Milestone 5: The "Traveler's Hub" (End of Phase 5)
- [ ] Public and Private group chats.
- [ ] Topic-based auto-discovery.
- [ ] High-speed Wi-Fi Direct file/media sharing.

### Milestone 6: "Production Ready" (End of Phase 6)
- [ ] **Privacy**: Incognito mode and Peer Blocking.
- [ ] **Performance**: Battery-intelligent discovery cycles.
- [ ] **Distribution**: Offline APK sharing via Wi-Fi.
- [ ] **Feel**: Tactile haptics and layered depth shadows.
